# QMT 团队框架详细流程

## 模式一：五步检查清单

```
[ ] 1. 分析：数据清单 + 来源
[ ] 2. 辩论：多空观点 + 批判性评估
[ ] 3. 决策：交易计划（可执行规格）
[ ] 4. 风险评估：四类风险 + 缓解措施
[ ] 5. 审批：风险调整后计划 + 实盘代码
```

## 分析阶段数据来源（QMT 函数）

| 数据类型 | QMT 函数 |
|----------|----------|
| K 线 OHLCV | get_market_data_ex |
| 换手率 | get_turnover_rate |
| 龙虎榜 | get_longhubang |
| 北向资金 | get_north_finance_change |
| 分笔/tick | get_full_tick |

## 风险评估四维度

1. **市场风险**：流动性不足、黑天鹅、波动率突变
2. **执行风险**：滑点、冲击成本、涨跌停无法成交
3. **模型风险**：过拟合、参数漂移、样本外失效
4. **资金风险**：最大回撤、单笔亏损、杠杆

## 模式二：回测优先铁律

- 第一步：仅写回测代码
- 第二步：用户确认回测通过
- 第三步：再写实盘代码（替换 order_value 为 passorder）
