# 资金流任务：约定与流程说明

本文档记录「每日资金流统计」相关的**所有约定**、**两个脚本的职责**及**数据流逻辑**，便于日后维护与防误删后的恢复。

---

## 一、输出文件约定

### 1.1 两个独立 Excel 文件

| 文件 | 用途 | 更新方式 |
|------|------|----------|
| **daily_fund_flow_wide.xlsx** | 每日资金流入汇总（宽表：每只股票一行，各交易日一列） | **持久化**：首次创建后，日常仅在**末尾追加当日一列**；历史空缺可由「补全历史脚本」单独填满。 |
| **fund_flow_summary_YYYYMMDD.xlsx** | 当日汇总（全量指标 + 前N日主力之和） | **按日新建**：每天运行「当日脚本」生成一份新文件，不覆盖前日。 |

- 每日表与汇总表**分离**的原因：每日表拉取/补全耗时长，且只需在末尾加一天；汇总表需要每天全新快照，便于按日存档与查看。

### 1.2 表结构约定

- **每日资金流入汇总**（daily_fund_flow_wide.xlsx 内）
  - 列：`股票`、`股票名称`，随后按交易日：`M/D主力`、`M/D散户`、`M/D涨跌幅`（如 `1/2主力`、`1/2散户`、`1/2涨跌幅`）。
  - 仅包含 **2026-01-01 起**的**交易日**，非交易日不建列。
- **汇总**（fund_flow_summary_YYYYMMDD.xlsx 内）
  - 列顺序见 excel_io（如：股票代码、股票名称、最新价、涨跌幅、当日净流入(万)、占成交金额、主力/散户净流入(万)、所属行业、流通市值(亿)、主力-散户净流入(万)、前3/5/10/20/40/60日主力净流入之和(亿)）。

---

## 二、单位与格式约定

- **汇总表**
  - 市值：**亿元**；流通市值(亿) 取整（整数）。
  - 资金流入：当日净流入、主力、散户、主力-散户 为 **万元**；**前 3/5/10/20/40/60 日主力净流入之和** 为 **亿元**。
  - 涨跌幅、占成交金额：**百分比字符串**，如 `"1.50%"`。
- **每日表**
  - `M/D主力`、`M/D散户`：**万元**。
  - `M/D涨跌幅`：**百分比字符串**，如 `"1.50%"`。
- **股票代码**
  - 表格中「股票代码」「股票」列一律为 **6 位文本**（如 `000001`），写入时 A 列设为文本格式并补零，避免 Excel 把前导 0 省略。

---

## 三、前 N 日主力净流入之和（数据准确性）

- **定义**：截止日（即汇总表文件名日期，如 20260310 表示 3/10）的**前** N 个交易日的**主力净流入**之和，**不含截止日当天**。
- **数据来源**：**仅**从「每日资金流入汇总」表中，按**股票代码**与**日期**取对应列的 `M/D主力`（单位已是万元）进行累加。
- **计算方式**：
  - 前 3 日 = 截止日的前 3 个交易日的 `M/D主力`（万元）列之和，再 ÷10000 得到亿元；
  - 前 5/10/20/40/60 日同理，取截止日的前 N 个交易日的 `M/D主力` 列（万元）相加，再换算为亿元写入。
  - **每只股票独立计算**：不同股票对应不同行，取各自行上对应日期列的值再求和，保证数据按股、按日一一对应，避免串行或混用。

汇总表生成时，由「当日运行脚本」读入当日最新的 daily_fund_flow_wide.xlsx，按上述规则计算前 3/5/10/20/40/60 日主力之和（亿元）后写入汇总表。

## 四、两个脚本的职责与关系

### 4.1 补全历史脚本（fill_daily_history）

- **脚本**：`qmt.fund_flow.fill_daily_history`（入口：`py -m qmt.fund_flow.fill_daily_history`）。
- **作用**：从 **2026 年年初（默认 2026-01-01）到指定截止日（含）**，对「每日资金流入汇总」表中**已有列**进行**按股、按日**补全：主力、散户、涨跌幅（单位：万元、%）。
- **使用场景**：
  1. **首次建表**：run_once 已生成带当日列的 daily_fund_flow_wide.xlsx 后，用本脚本把历史区间内的空列填满。
  2. **信息误删或丢失**：若每日表被误删或历史列被清空，可先再跑一次 run_once 生成带当日列的宽表，再运行本脚本，重新从 2026 年年初补全到当天，**不依赖当日脚本的历次运行记录**。
- **数据源**：当前实现使用 akshare 按股拉取历史资金流；历史与当日东方财富口径可能略有差异，属正常。
- **参数示例**：
  - `--path output/daily_fund_flow_wide.xlsx`（默认）
  - `--end 2026-03-10`（截止日）
  - `--start 2026-01-01`（起始日）
  - `--limit N`（仅处理前 N 只股票，用于测试）

**重要**：本脚本为**单独补全历史**用，与「每天跑一次的当日脚本」解耦；只要每日表结构在（可由 run_once 生成），即可随时用本脚本把 2026 年年初到当天的历史数据补全，保证后期当日脚本只关心「追加当日」和「从每日表累加前 N 日」。

### 4.2 当日运行脚本（run_once）

- **脚本**：`qmt.fund_flow.run_once`（入口：`py -m qmt.fund_flow.run_once`）。
- **作用**：
  1. 拉取**当日**全 A 股资金流与行情（东方财富），清洗后得到当日主力/散户净流入（万元）等。
  2. **每日表**：若 `daily_fund_flow_wide.xlsx` 不存在，则创建并写入当日列；若已存在，则**仅在末尾追加当日**一列（M/D主力、M/D散户、M/D涨跌幅），不重写历史列。
  3. **汇总表**：生成 `fund_flow_summary_YYYYMMDD.xlsx`；其中**前 3/5/10/20/40/60 日主力净流入之和(亿)**从**当前 daily_fund_flow_wide.xlsx** 的「每日资金流入汇总」中，按股票、按对应交易日列**累加**（万元）后除以 10000 得到亿元（不含当日），保证不同股票对应不同数据、按日累加准确。
- **使用场景**：每日收盘后（如 16:00）定时或手动执行一次，用于「当日数据汇总」与「当日汇总表」的生成；不负责历史区间的补全。
- **截止日**：支持命令行 `--date`，可传「今天」「昨天」或 `YYYY-MM-DD`（如 `2026-03-11`）；不传则默认跑**今天**。

#### 首次运行无历史时，前 N 日为 0；补全历史后再次运行

- **第一次运行 run_once**：若此时 `daily_fund_flow_wide.xlsx` 不存在或历史列为空，汇总表里的「前 3/5/10/20/40/60 日主力净流入之和(亿)」没有可累加的历史列，**结果会为 0 或空**。
- **补全历史**：运行 `fill_daily_history` 后，每日表中从 2026 年年初到当天的列已被填满。
- **第二次运行 run_once**：
  - **会**从已补全的每日表重新读取并累加，生成汇总表时**前 N 日会变为正确数值**（不再全为 0）。
  - **是否覆盖**：汇总表文件名按 `snapshot_date` 生成（如 `fund_flow_summary_20260310.xlsx`）。
    - 若第二次运行的**截止日与第一次相同**（例如仍为 3/10），则写入**同一文件名**，会**覆盖**该日期的汇总表，得到前 N 日正确的那一版。
    - 若第二次运行是**次日**（如 3/11），则生成的是新文件 `fund_flow_summary_20260311.xlsx`，**不会覆盖** 3/10 的汇总表；之前 3/10 那份仍保留第一次的「前 N 日 = 0」。若希望 3/10 的汇总表也带上正确前 N 日，需用**同一截止日**（如 `snapshot_date=2026-03-10`）再跑一次 run_once，会覆盖 `fund_flow_summary_20260310.xlsx` 并写入正确前 N 日。

### 4.3 数据流关系（保证数据准确性）

```
补全历史脚本（按需运行）
  从 2026-01-01 到当天 → 按股、按日写入/补全「每日资金流入汇总」中的 M/D主力、M/D散户、M/D涨跌幅
        ↓
daily_fund_flow_wide.xlsx（持久化）
  每行 = 一只股票；每列 = 某交易日的主力/散户/涨跌幅（万元、%）
        ↓
当日运行脚本（每日运行）
  1) 拉取当日数据 → 追加一列到 daily_fund_flow_wide.xlsx
  2) 读取 daily_fund_flow_wide.xlsx → 按股票、按前 N 个交易日列求和（万元）→ ÷10000 得亿元 → 写入汇总表「前3/5/10/20/40/60日主力净流入之和(亿)」
  3) 生成 fund_flow_summary_YYYYMMDD.xlsx
```

- **不同股票对应不同数据**：汇总表每一行的前 N 日之和，只使用每日表中**该股票那一行**的对应日期列，不做跨行或全局汇总，保证按股准确。
- **日期对应**：前 N 日严格按「交易日序列」取前 N 个交易日（不含当日），与 calendar_utils 的交易日历一致。

## 五、其他数据获取方式（当东方财富直连/浏览器分页失败时）

当 `run_once` 直连东方财富失败、且浏览器手动分页保存（page4 及以后）也打不开时，可选用以下方式之一：

| 方式 | 脚本 / 操作 | 说明 |
|------|--------------|------|
| **1. 已保存的 JSON** | `py -m qmt.fund_flow.run_once_from_json --input qmt/fund_flow/output --date 2026-03-11` | 将浏览器能打开的前几页（如 page1～page3）复制保存为 `page1.json`、`page2.json`… 放入 `output`，本脚本会合并后生成汇总与每日宽表。 |
| **2. 仅 akshare** | `py -m qmt.fund_flow.run_once_akshare_only --date 2026-03-11` | 不请求东方财富 API，仅用 akshare 的「今日个股资金流排名」拉取；**最多约 200 条**，输出格式与 run_once 一致。适合 akshare 能通、直连不通的环境。 |
| **3. 换网络 / 换时段** | 稍后再试 `run_once` 或浏览器分页保存 | 东方财富有时对 IP/时段限流，换网络（如手机热点）或非高峰再试可能恢复。 |

- 方式 1：数据量取决于你手动保存的页数（每页约 100 条）。
- 方式 2：数据量固定约 200 条，为当日主力净流入排名靠前的股票。

## 六、行业、模板与其它约定

- **行业**：所属行业来自同花顺 + 东方财富行业成份合并（先同花顺再东财补漏），减少缺失；若仍为空多为新上市或未纳入板块。
- **模板**：若在 output 目录下放置 `fund_flow_summary_template.xlsx`、`daily_fund_flow_wide_template.xlsx`，写入汇总表与每日表时会优先套用模板格式（字体、颜色、列宽等），仅更新数据与代码列格式。
- **全量历史补全完成**：运行 fill_daily_history 全量补全（无 `--limit`）结束后，终端会打印「【全量历史补全已完成】」；见到即表示历史已写入并保存成功。

## 七、一次性建表 vs 日常更新（速查）

| 场景 | 操作 |
|------|------|
| **首次建表 / 误删后重建** | 1) `py -m qmt.fund_flow.run_once` → 生成 daily_fund_flow_wide.xlsx（当日列有数据）+ 当日汇总表（此时前 N 日多为 0）<br>2) `py -m qmt.fund_flow.fill_daily_history`（可选 `--end` 为今天）→ 从 2026 年年初补全到当天<br>3) 若希望**同一天**的汇总表前 N 日也正确：用**同一 snapshot_date** 再跑一次 `run_once`，会覆盖该日期的汇总文件并写入正确前 N 日 |
| **日常更新（之后每天）** | 仅运行 `py -m qmt.fund_flow.run_once`：追加当日列到每日表，并生成当日 fund_flow_summary_YYYYMMDD.xlsx（前 N 日从每日表累加）。 |
