# 数据来源、已保存 JSON 字段说明与代理设置

## 一、已保存的 JSON（page1/2/3）里有什么数据

你从浏览器复制保存的东方财富接口返回的 JSON，当前 URL 的 **fields** 为：

`f12,f14,f2,f3,f20,f21,f62,f184,f6`

对应含义与用途如下：

| 字段 | 含义       | 汇总表/宽表用途     |
|------|------------|----------------------|
| f12  | 股票代码   | 股票代码列           |
| f14  | 股票名称   | 股票名称列           |
| f2   | 现价       | 最新价               |
| f3   | 涨跌幅(%)  | 涨跌幅               |
| f6   | 成交额(元) | 占成交金额% 计算     |
| f20  | 总市值(元) | 可转亿元展示         |
| f21  | 流通市值(元) | 流通市值(亿)       |
| f62  | 主力净流入(元) | 主力净流入(万)、前N日之和 |
| f184 | 小单/散户净流入相关 | 散户净流入(万)   |

**没有包含在现有 URL 里的：**

- **所属行业**：需单独接口（akshare/东财行业）或本地 `industry_map.csv`，或在请求 URL 里增加行业相关字段（若接口支持，如 f127 等）后再保存 JSON，脚本会从 JSON 里解析。

所以：**当前这份 JS/JSON 已经能提供做「当日汇总」所需的代码、名称、现价、涨跌幅、成交额、总市值、流通市值、主力净流入、散户净流入**；行业需要另补（见上文或《约定与流程》里行业说明）。

## 二、同花顺、通达信及其他接口

- **同花顺**
  - 官方：需用 **iFinD / 同花顺量化接口**（申请账号），如 `iFinDPy` 的 `THS_WC('主力资金流向','stock')` 等，非开放免费 HTTP 接口。
  - 爬虫：同花顺网页有反爬（如 hexin 等），需处理 Cookie/加密，维护成本高，且易失效。
  - akshare 里同花顺相关（如行业板块）底层也是网络请求，你当前环境若被限流，同样可能失败。

- **通达信**
  - 主要是**本地客户端**（通达信软件），没有公开的、可直接用 Python 调用的 HTTP 资金流 API；量化多用其本地接口或 TGP 等，不适合本脚本这种「远程拉一份全市场资金流」的用法。

- **其他可选数据源**
  - **Tushare**：有资金流等数据，需注册拿 token，部分接口需积分/付费。
  - **akshare**：当前脚本已用其作东方财富的备用（今日个股资金流排名），约 200 条；和你直连东财一样受网络影响。
  - **第三方付费 API**（如部分 stockapi 等）：可按需比较价格与字段，再对接。

结论：**在不买付费接口、不爬同花顺网页的前提下**，继续用「东方财富接口 + 浏览器保存 JSON」或「走代理/VPN 的东财/akshare」是最现实的做法；同花顺/通达信没有像东财这种「一个 URL 返回全市场资金流」的免费替代。

## 三、如何“伪装 IP”或换出口——用代理拉取

东方财富对部分 IP/地区会限流或断开，换一个出口 IP 往往就能正常拉取。可以用**代理（Proxy）**或 **VPN**，让脚本的请求从代理/VPN 出去。

### 1. 环境变量设置代理（推荐）

在运行脚本的终端里先设置环境变量，再执行命令（一次会话有效）：

**Windows (CMD)：**

```bat
set HTTP_PROXY=http://127.0.0.1:7890
set HTTPS_PROXY=http://127.0.0.1:7890
py -m qmt.fund_flow.run_once --date 2026-03-11
```

**Windows (PowerShell)：**

```powershell
$env:HTTP_PROXY="http://127.0.0.1:7890"
$env:HTTPS_PROXY="http://127.0.0.1:7890"
py -m qmt.fund_flow.run_once --date 2026-03-11
```

**说明：**

- `127.0.0.1:7890` 改成你自己本机的代理地址和端口（如 Clash、V2Ray、SSR 等常用 7890/7891/1080）。
- 用 VPN 时，若 VPN 提供的是「系统代理」，一般也是本机某个端口，填该端口即可。
- 脚本会读取 `HTTP_PROXY` / `HTTPS_PROXY`（或小写 `http_proxy` / `https_proxy`），**直连拉取**（`fetcher`）和**浏览器方式拉页**（`fetch_pages_to_json_browser`）都会走该代理。

### 2. 哪些脚本会走代理

- **run_once**：内部用 `fetcher` 请求东方财富，会走上述环境变量代理。
- **fetch_pages_to_json_browser**：Playwright 打开浏览器访问东财 URL，也会读同一套环境变量并走代理。
- **run_once_from_json**：只读本地 JSON，不发起对东财的请求，**不需要**代理。
- **export_industry_map**：拉取行业时会请求网络，设置代理后也会走代理。

### 3. 代理仍不通时

- 换一个代理节点或 VPN 节点（换国家/地区）。
- 用手机开热点，电脑连热点再跑（换出口 IP）。
- 换时间段再试（避开高峰）。
- 继续用「浏览器手动保存 page1～pageN.json + run_once_from_json」做汇总，不依赖脚本直连。

## 四、小结

| 问题 | 简要回答 |
|------|----------|
| 已保存的 JS/JSON 能获取什么数据？ | 代码、名称、现价、涨跌幅、成交额、总市值、流通市值、主力净流入、散户相关；**不含行业**（需另补或 URL 加字段）。 |
| 能用同花顺/通达信接口吗？ | 同花顺需官方量化接口或爬虫（成本高）；通达信无此类开放 HTTP 资金流 API；暂无直接替代。 |
| 怎么“伪装 IP”让网站开放拉取？ | 本机开代理/VPN，设置 **HTTP_PROXY / HTTPS_PROXY** 后运行 run_once 或 fetch_pages_to_json_browser，请求会从代理出口发出。 |
