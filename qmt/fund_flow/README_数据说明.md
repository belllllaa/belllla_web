# 资金流 Excel 数据说明

> 约定、脚本分工与流程见 **README_约定与流程.md**（两文件、补全历史脚本 vs 当日脚本、前 N 日累加逻辑）。

## 单位

- **汇总表**：市值单位为 **亿元**（流通市值取整）。资金流入：当日净流入、主力、散户、主力-散户 为 **万元**；**前 3/5/10/20/40/60 日主力净流入之和** 为 **亿元**。涨跌幅、占成交金额为百分比字符串（如 `"1.50%"`）。
- **每日资金流入汇总**：`M/D主力`、`M/D散户` 单位均为 **万元**；`M/D涨跌幅` 为百分比字符串（如 `"1.50%"`）。

## 数据来源与一致性

- **汇总表**：当日数据来自东方财富接口（run_once 爬取），**当日净流入(万) = 主力净流入(万) + 散户净流入(万)**。
- **每日 sheet 的「当日」列**：与汇总表同源（同一次 run_once 写入），故 **当日净流入(万) = 当日列主力 + 当日列散户**，应与汇总表一致。
- **每日 sheet 的历史列**（非当日的 M/D 列）：由 `fill_daily_history` 用 akshare 按股补全（含主力、散户、涨跌幅），与东方财富口径可能略有差异，属正常。

## 前 3/5/10/20/40/60 日主力净流入之和

- **定义**：截止日（文件名日期，如 20260309 表示 3/9）的**前** N 个交易日的主力净流入之和，**不含截止日当天**。N 取 3、5、10、20、40、60。
- **举例**：若截止日为 **3/10**，则  
  - **前5日** = 3/3 + 3/4 + 3/5 + 3/6 + 3/9 的「主力」列之和（5 个交易日）；  
  - 若截止日为 **3/9**，则前5日 = 3/2 + 3/3 + 3/4 + 3/5 + 3/6。

计算时以「每日资金流入汇总」中对应日期的 `M/D主力` 列（**万元**）相加，再 **÷10000** 得到亿元，写入汇总表「前3/5/10/20/40/60日主力净流入之和(亿)」。

### 首次运行无历史时前 N 日为 0；补全后再次运行

- **第一次运行 run_once** 时，若每日表尚无历史列（或刚创建），汇总表里的前 3/5/10/20/40/60 日主力之和(亿)**会为 0 或空**（没有可累加的历史列）。
- 运行 **fill_daily_history** 补全每日表后，再运行 **run_once**：会从已补全的每日表重新累加，生成的汇总表**前 N 日会变为正确数值**。
- **是否覆盖**：汇总表按截止日命名（如 `fund_flow_summary_20260310.xlsx`）。若用**同一截止日**再跑一次 run_once，会**覆盖**该日期的汇总文件并写入正确前 N 日；若跑的是**次日**，则生成新文件，不会改前一天的汇总表（前一天那份仍为首次的“前 N 日=0”，除非用同一截止日再跑一次）。

## 股票代码格式

- 表格中 **股票代码**、**股票** 列已强制为 **6 位文本**（如 `000001`），避免 Excel 把 `000001` 显示成 `1`。写入时 A 列设为文本格式并统一补零；若仍见 00 开头被省略，请重新跑一次 `run_once` 生成新文件。

## 全量历史补全完成提示

- 运行 `fill_daily_history`（无 `--limit`）全量补全时，**结束后终端会打印**：
  ```
  ==================================================
  【全量历史补全已完成】
    共处理 xxxx 只股票，已保存至: ...\fund_flow_YYYYMMDD.xlsx
    每日 sheet 已补全历史主力/散户/涨跌幅（单位：万元、%）。
  ==================================================
  ```
- 看到上述提示即表示全量历史补全已跑完并保存成功。

## 一次性建表 vs 日常更新

- **一次性（首次或误删后重建）**：见 **README_约定与流程.md** 第四节。
  1. 运行 `py -m qmt.fund_flow.run_once` → 创建/更新 `output/daily_fund_flow_wide.xlsx`（当日列有数据），并生成当日 `fund_flow_summary_YYYYMMDD.xlsx`（此时前 N 日多为 0）。
  2. 运行 `py -m qmt.fund_flow.fill_daily_history`（可加 `--end` 为今天）→ 从 2026 年年初补全到当天的主力/散户/涨跌幅（万元、%），写回每日表。
  3. 若希望**同一天**的汇总表前 N 日也正确：用**同一 snapshot_date** 再跑一次 `run_once`，会覆盖该日期的汇总文件并写入正确前 N 日。
- **日常更新（之后每天）**：只运行 `py -m qmt.fund_flow.run_once`：追加当日列到每日表，并从每日表按股票、按日期累加前 3/5/10/20/40/60 日主力（万元）再换算为亿元写入当日汇总表。

## 行业

- **所属行业** 来自同花顺行业 + 东方财富行业成份合并（先同花顺，再东财补漏），以减少行业缺失。若个别标的仍为空，多为新上市或分类未纳入板块。
