- **要什么字段**：见 [二、表 1](#二表-1每日资金流入汇总宽表) 与 [三、表 2](#三表-2当日汇总按日快照) 的列名、类型、单位。
- **源在哪里**：见 [五、数据源在哪里](#五数据源在哪里接口与-url)（东方财富 push2 接口 URL、参数、行业来源）。
- **计算逻辑**：见 [六、源字段→目标字段与计算逻辑](#六源字段--目标字段与计算逻辑)（单位换算、当日净流入、占成交金额、前 N 日主力之和等）。

## 一、需求概览

| 数据类型 | 用途 | 更新方式 |
|----------|------|----------|
| **每日资金流入汇总（宽表）** | 每只股票一行，各交易日一列（主力/散户/涨跌幅），用于按日查询与计算前 N 日主力之和 | 持久化一张表，每日**追加当日一列**（或新增当日一行若按日期+股票为主键） |
| **当日汇总（按日快照）** | 每个交易日一份快照：全市场股票的代码、名称、最新价、涨跌幅、当日净流入、主力/散户、所属行业、流通市值、前 3/5/10/20/40/60 日主力净流入之和等 | 按日生成/覆盖，**每自然日或交易日一条记录**（或按 日期+股票代码 为主键的明细表） |

---

## 二、表 1：每日资金流入汇总（宽表）

- **表名建议**：`daily_fund_flow_wide` 或等价。
- **主键**：`股票`（6 位股票代码，字符串）。
- **维度**：一行 = 一只股票；列 = 固定维度 + 各交易日的 3 列（主力、散户、涨跌幅）。

### 2.1 固定列（每行必有）

| 列名 | 类型 | 说明 |
|------|------|------|
| 股票 | varchar(6) / 文本 | 6 位股票代码，前导补零（如 000001） |
| 股票名称 | varchar | 股票名称 |

### 2.2 按交易日动态列（仅包含交易日，如 2026-01-01 起）

每个交易日占 **3 列**，列名格式：`{M/D}主力`、`{M/D}散户`、`{M/D}涨跌幅`。  
其中 `M/D` 为「月/日」，例如：`1/2`、`3/9`（即 1月2日、3月9日）。

| 列名示例 | 类型 | 单位/格式 |
|----------|------|-----------|
| 1/2主力 | decimal | **万元** |
| 1/2散户 | decimal | **万元** |
| 1/2涨跌幅 | varchar | 百分比字符串，如 `"1.50%"` |

- **日期范围**：仅 2026-01-01 起的**交易日**，非交易日不建列。
- **更新逻辑**：每日在表上**追加**「当日」对应的 3 列（如 `3/11主力`、`3/11散户`、`3/11涨跌幅`）；若表结构为「长表」（日期+股票为行），则每日追加当日所有股票的一批行。

## 三、表 2：当日汇总（按日快照）

- **表名建议**：`fund_flow_summary` 或按日分表 `fund_flow_summary_YYYYMMDD`。
- **主键**：`(日期, 股票代码)` 或 按文件日期 + 股票代码。

### 3.1 字段列表（列顺序建议保持一致）

| 列名 | 类型 | 单位/说明 |
|------|------|-----------|
| 股票代码 | varchar(6) | 6 位，前导补零 |
| 股票名称 | varchar | |
| 最新价 | decimal | 元 |
| 涨跌幅 | varchar | 如 `"1.50%"` |
| 当日净流入(万) | decimal | **万元**（= 主力 + 散户） |
| 占成交金额 | varchar | 如 `"2.30%"`，可为空 |
| 主力净流入(万) | decimal | **万元** |
| 散户净流入(万) | decimal | **万元** |
| 所属行业 | varchar | 可为空 |
| 流通市值(亿) | decimal / int | **亿元**，建议取整 |
| 主力-散户净流入(万) | decimal | **万元** |
| 前3日主力净流入之和(亿) | decimal | **亿元** |
| 前5日主力净流入之和(亿) | decimal | **亿元** |
| 前10日主力净流入之和(亿) | decimal | **亿元** |
| 前20日主力净流入之和(亿) | decimal | **亿元** |
| 前40日主力净流入之和(亿) | decimal | **亿元** |
| 前60日主力净流入之和(亿) | decimal | **亿元** |

- **前 N 日主力净流入之和**：截止日（即该快照日期）的**前** N 个**交易日**的「主力净流入」之和，**不含截止日当天**；数据来源为「每日资金流入汇总」表中对应日期的 `M/D主力` 列（万元），累加后换算为**亿元**写入。

---

## 四、前 N 日主力之和的计算规则（供 ETL/视图用）

- **前 3 日**：截止日的前 3 个交易日的 `M/D主力`（万元）相加，再 ÷10000 得到亿元。
- **前 5 / 10 / 20 / 40 / 60 日**：同理，取截止日的前 N 个交易日的 `M/D主力` 列（万元）相加，再换算为亿元。
- **每只股票独立计算**：按股票代码取该行在每日表中的对应列，再按日期维度取「前 N 个交易日」的列值求和。

## 五、数据源在哪里（接口与 URL）

### 5.1 主数据源：东方财富 push2 个股资金流列表

- **接口地址**：`https://push2.eastmoney.com/api/qt/clist/get`
- **请求方式**：GET
- **常用请求参数**（我方实际使用）：

| 参数 | 值 | 说明 |
|------|-----|------|
| fs | m:0+t:6,m:0+t:80,m:1+t:2,m:1+t:23 | 全 A 股（沪主板+科创板+深主板+创业板） |
| fid | f62 | 按主力净流入排序 |
| po | 1 | 排序方向 |
| pz | 100～500 | 每页条数（接口常限制约 100，需分页） |
| pn | 1, 2, 3… | 页码 |
| np | 1 |  |
| fltt | 2 |  |
| invt | 2 |  |
| **fields** | **f12,f14,f2,f3,f20,f21,f62,f184,f6** | 必填，顺序与返回 diff 中每条记录的数组下标对应 |

- **返回结构**：`data.data.diff` 为数组，每项为**按 fields 顺序的数组**或对象；若为数组，下标与 fields 一致：
  - `[0]`=f12 代码，`[1]`=f14 名称，`[2]`=f2 现价，`[3]`=f3 涨跌幅，`[4]`=f20 总市值，`[5]`=f21 流通市值，`[6]`=f62 主力净流入，`[7]`=f184 散户净流入，`[8]`=f6 成交额。
- **单位（接口原始）**：价格/市值/成交额/资金流均为**元**；涨跌幅为**数值**（如 1.5 表示 1.5%）。

### 5.2 行业数据源（所属行业）

- **所属行业**不在上述 push2 的 fields 中，需单独获取：
  - **同花顺行业**：akshare `stock_board_industry_summary_ths` + 各板块 `stock_board_industry_cons_ths(symbol=板块名)`，得到「代码 + 所属行业」；
  - **东方财富行业**：akshare 东财行业成份接口补全同花顺未覆盖的代码；
  - 合并规则：同花顺优先，东财填漏；若仍无则留空。

## 六、源字段 → 目标字段与计算逻辑

### 6.1 源字段含义与单位（东方财富 push2，fields=f12,f14,f2,f3,f20,f21,f62,f184,f6）

| 源字段 | 含义 | 接口单位 | 说明 |
|--------|------|----------|------|
| f12 | 股票代码 | 文本 | 需格式化为 6 位字符串，前导补零（如 000001） |
| f14 | 股票名称 | 文本 | 直接使用 |
| f2 | 现价 | 元 | → 最新价（元） |
| f3 | 涨跌幅 | 数值（如 1.5 表示 1.5%） | → 涨跌幅，展示为 "1.50%" 等 |
| f20 | 总市值 | 元 | 可选，汇总表未用 |
| f21 | 流通市值 | 元 | → 流通市值(亿)：见下方换算 |
| f62 | 主力净流入 | 元 | → 主力净流入(万)：见下方换算 |
| f184 | 小单/散户净流入 | 元 | → 散户净流入(万)：见下方换算 |
| f6 | 成交额 | 元 | 用于计算「占成交金额」 |

### 6.2 单位换算规则（ETL 时必须一致）

- **元 → 万元**（主力净流入、散户净流入、以及每日表中的 M/D主力、M/D散户）：
  - 若 `abs(值) >= 10000`：`万元 = 值 / 10000`，保留 2 位小数；
  - 若已像万元量级（`abs(值) < 10000`）：可直接按原值保留 2 位小数（避免重复除）。
- **元 → 亿元**（流通市值、总市值）：
  - 若 `值 >= 10000`：`亿元 = 值 / 1e8`，保留 4 位小数；汇总表「流通市值(亿)」再取整为整数；
  - 若已像亿元量级：可直接按原值。
- **涨跌幅**：接口为数值（如 1.5），展示为字符串 `"1.50%"`（保留 2 位小数）。

### 6.3 汇总表各字段的计算逻辑

| 目标字段 | 计算逻辑 |
|----------|----------|
| 股票代码 | 源 f12，格式化为 6 位字符串，前导补零 |
| 股票名称 | 源 f14 |
| 最新价 | 源 f2（元），直接写入 |
| 涨跌幅 | 源 f3 → 格式化为 "x.xx%" |
| 当日净流入(万) | **主力净流入(万) + 散户净流入(万)**（换算后的万元） |
| 占成交金额 | **当日净流入(万) × 10000 / 成交额(元) × 100**，格式化为 "x.xx%"；成交额为 0 时为空 |
| 主力净流入(万) | 源 f62，按 6.2 换算为万元 |
| 散户净流入(万) | 源 f184，按 6.2 换算为万元 |
| 所属行业 | 见 5.2，单独接口或 industry_map 映射 |
| 流通市值(亿) | 源 f21，按 6.2 换算为亿元后**取整** |
| 主力-散户净流入(万) | **主力净流入(万) − 散户净流入(万)** |
| 前3/5/10/20/40/60日主力净流入之和(亿) | 见下方 6.4 |

### 6.4 前 N 日主力净流入之和（亿元）的计算逻辑

- **数据来源**：表 1「每日资金流入汇总」中，该股票行在**截止日之前**的 N 个**交易日**对应的 `M/D主力` 列（单位已是**万元**）。
- **计算步骤**：
  1. 确定截止日（即汇总表快照日期）及该日之前的交易日序列（仅交易日，不含节假日）；
  2. 取截止日**前** N 个交易日（不含截止日当天），如 N=5 则取前 5 个交易日的 5 列；
  3. 对该股票在该 N 个交易日列上的 `M/D主力`（万元）**求和**；
  4. **万元 → 亿元**：`前N日主力之和(亿) = 上述求和 / 10000`，保留 4 位小数。
- **示例**：截止日为 3/10，前 5 日 = 3/3、3/4、3/5、3/6、3/9 的「M/D主力」列（万元）相加，再除以 10000 得到亿元。
