# 卡异动实盘三策略要点

> 本文档与 `strategy_yidong_*_live.py` 三份源码一致；分档买卖规则见下文「回测/入池参考」，**尾盘卖策略不会按 F 档自动改持有日或卖出日**。

| 文件 | 策略 |

| `strategy_yidong_close_buy_live.py` | 尾盘买 |

三个文件互相独立，不 import 公共模块。

## QMT 通用设置

- 周期：**1 分钟**
- 默认资金账号：**30262698**（可在 QMT `ContextInfo.accountid` 覆盖）
- 买入单票上限：**10 万元**（`max_amount_per_stock`，默认 100000）
- 买卖股数：**100 股整数倍**（`min_shares`，默认 100）
- 接口从 `__main__` 取：`passorder` / `cancel` / `get_trade_detail_data`
- 入口：`init` + `handlebar`（别名 `handleBar` / `handle_bar`）
- 仅 `C.is_last_bar()` 时执行
- 回测不发单（`live_orders=False` 或回测上下文）

### 可选 ContextInfo 参数（通用）

| 参数 | 默认 | 说明 |
|------|------|------|
| `live_orders` | True | False 只打日志不下单 |
| `max_retry` | 3 | 撤单/废单后重挂最多次数（首单之外最多再挂 3 次） |
| `quick_trade` | 2 | passorder 快速交易 |

> **撤单等待**：买策略默认 `withdraw_secs=15`；卖策略默认 `withdraw_secs=60`。可在各自策略参数中覆盖。

## 回测/入池参考：F 档分档规则

以下规则在 `qmt/scripts/_yidong_final_rules_backtest.py` 与上游 CSV 筛选用；**三个实盘脚本不读取 F 档，卖出也不按 F 自动分支**。入池、监控表需人工或外部流程按规则准备。

| 条件 | 买入 | 卖出（收盘） | 说明 |
|------|------|--------------|------|
| F ∈ {1,2,3,4,5,6} | 不买 | — | 同 T 日+代码整键剔除 |
| F7 | T+2 开盘 | T 日收涨>0 → T+3 收；T 日收跌/平≤0 → T+4 收 | 分支按 **T 日收盘相对 T-1 收盘** |
| F10 | T+1 开盘 | T 日收涨>0 → T+2 收；T 日收跌/平≤0 → T+3 收 | 同上 |
| F8 / F30 | T+1 开盘 | T+4 收盘 | 固定持 3 个交易日 |
| F9 / F27 / F28 | T+1 开盘 | T+3 收盘 | 固定持 2 个交易日 |
| 其余 F | T 日开盘 | T+2 起 8% 止损 + T+6 兜底 | 与尾盘卖默认参数对齐 |
| 全档 | — | 16% 止盈 | 回测与实盘 `take_profit_pct=16` |

信号池涨跌幅过滤（上游 `is_limit_skip_row`）：T 日收盘相对 T-1，60/00 跌幅带 [-10%,-8%] 或涨幅>8%；68/30 跌幅带 [-30%,-15%] 或涨幅>15% → 不入池。

**实盘与回测的差异**：开盘买/尾盘买在下单时刻用 **现价/昨收** 做涨跌幅过滤（见下节），不是 T 日收盘。

## 策略1：开盘买

- **handlebar**：09:25–11:30、13:00–15:00（午休 11:30–13:00 不跑）
- **实际下单**：09:25–09:35
- **集合竞价补发**：实盘默认 `auction_timer=True`，`run_time` 自 09:25 每 5 秒触发 `on_open_buy_timer`（1 分钟 K 首根常在 9:30，否则 9:25 无法扫池）
- **日志**：板块成分变化时打 `[板块重载]` + `[今日待买池]`（名称+代码）
- **股票池**：板块 `卡异动开盘买入池`（`sector_name` 可覆盖）；**24 小时**每根 K 线重拉板块；**仅成分变化**时打日志
- **单票上限**：10 万，100 股整数倍；**不因已有持仓扣减额度**（允许加仓）
- **涨跌幅过滤**（现价 / 昨收，下单时刻）：
  - 60/00：跌幅 [-10%, -8%] 或涨幅 >8% → 不买
  - 68/30：跌幅 [-30%, -15%] 或涨幅 >15% → 不买
- **限价**：现价 × (1 + **0.5%**)，`buy_premium_pct` 默认 **0.005**
- **未成交处理**：**15 秒**后检查；挂单未成交则撤单重挂；**废单/已撤也会重挂**；最多 `max_retry` 次
- **重挂溢价**：`buy_premium_retry_step` 默认 **0**（重挂不额外加价）
- **备注前缀**：`YDO_*`（最长 23 字符）

### 可选参数（开盘买）

| 参数 | 默认 |
|------|------|
| `max_amount_per_stock` | 100000 |
| `buy_premium_pct` | **0.005** |
| `buy_premium_retry_step` | 0 |
| `withdraw_secs` | 15 |
| `sector_name` | 卡异动开盘买入池 |
| `auction_timer` | True |
| `auction_timer_period` | 5nSecond |

## 策略3：尾盘卖

- **handlebar**：09:25–11:30、13:00–15:00（午休不跑）
- **盘中**：每根 K 线仅评估 **16% 止盈**（**09:30–15:00**，限价 = 止盈价）
- **尾盘卖出窗口**：14:55–15:00 评估 **全部卖出条件** 并下单
- **数据源**：CSV + 账户持仓（不用板块）
- **CSV 24h 监控**：每根 K 线重读；**仅 CSV 内容变化**时打 `[CSV重载]` + `[今日待卖池]`；持仓变化时打 `[账户持仓]`
- **卖出窗口内**：每根 K 线打 `[监控]`（含持仓交易日、盈亏、是否触发）
- **CSV 路径**（按优先级）：
  1. `ContextInfo.sell_watch_csv_path`
  2. 环境变量 `YIDONG_SELL_WATCH_CSV`
  3. 代码内默认绝对路径：`C:\Users\Dustin.hou\belllla_web\qmt\实盘策略\yidong_regulation_sell_watch.csv`
  4. 策略同目录 `yidong_regulation_sell_watch.csv`
  5. 从 QMT 工作目录（常为 `bin.x64`）向上查找
- **CSV 列**：`股票代码, 开仓日, 触发卖出信号, 卖出股数, 开仓价`（可选第 6 列止盈价）
- **编码**：utf-8-sig / gbk / utf-8 自动尝试

### 卖出触发

| 条件 | 评估时段 | 说明 |
|------|----------|------|
| **止盈** | 盘中 **09:30–15:00** + 尾盘 | 现价 ≥ 止盈价；止盈价 = CSV 第 6 列，或 `开仓价 × (1 + take_profit_pct%)`（默认 16%） |
| 触发卖出信号 = **0** | 仅 14:55–15:00 | G0 强平 |
| 持仓交易日 **≥ 6** | 仅 14:55–15:00 | `max_hold_days` 默认 6（相对开仓日，节假日不计） |
| 持仓交易日 **≥ 2** 且浮亏 **≥ 8%** | 仅 14:55–15:00 | `stop_min_hold_days=2`（**T+2 起**）；基准价 **优先 CSV 开仓价**，无则账户成本 |

> 信号 = **1** 时不会因 G0 卖出，但止盈、满 6 交易日、止损仍会触发。止损日志示例：`止损8%(开仓价X.XXX持N日,浮亏-8.XX%)`。

### 下单规则

- 数量：优先读 CSV **卖出股数**（按原数，**不必整百**）；取 `min(CSV股数, 可卖)`；旧表无该列则卖出全部可卖
- CSV 股数 ≤ 0 或无效 → 跳过不卖
- **限价**：一般现价 × (1 - 1%)，`sell_discount_pct` 默认 0.01；**止盈单限价 = 止盈价**
- **60 秒**未成交撤单重挂，废单也会重挂，最多 `max_retry` 次
- **备注前缀**：`YDS_*`

### 可选参数（卖策略）

| 参数 | 默认 | 说明 |
|------|------|------|
| `take_profit_pct` | **16** | 无 CSV 止盈价时，按开仓价计算 |
| `stop_loss_pct` | 8 | 相对止损基准价的跌幅 |
| `stop_min_hold_days` | **2** | 开仓后第 2 个交易日起才检查止损（T+2 起） |
| `max_hold_days` | 6 | 持满兜底卖出 |
| `sell_discount_pct` | 0.01 | 非止盈卖单折价 |
| `withdraw_secs` | 60 | 撤单等待 |

## CSV 卖出监控表示例

```csv
股票代码,开仓日,触发卖出信号,卖出股数,开仓价
600000.SH,20250601,1,1000,10.00
000001.SZ,20250601,0,500,8.50
688143.SH,20260529,1,636,10.75
```

- 信号 = **0**：当日尾盘 G0 强平（若可卖）
- 信号 = **1**：继续持有，直至止盈、满 6 交易日或触发止损
- **卖出股数**：本笔买入对应数量；同代码多笔请分多行；不填则卖全部可卖
- **开仓价**：该笔实际买入成交价；用于 **16% 止盈** 与 **8% 止损** 基准（优先于账户成本）
- **可选第 6 列止盈价**：若填写则覆盖 `开仓价×16%` 自动计算；**现价 ≥ 止盈价** 即卖，限价 = 止盈价
- 策略参数 `ContextInfo.take_profit_pct` 可改止盈比例（默认 16）
- **表格编辑**：用 VS Code / Excel 打开时，**D 列=卖出股数、E 列=开仓价**；股数与开仓价分列填写，勿在股数后加逗号
- CSV 内仅保留表头与数据行，便于表格视图显示
