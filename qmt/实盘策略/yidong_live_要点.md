# 卡异动实盘三策略要点

## 文件

| 文件 | 策略 |
|------|------|
| `strategy_yidong_open_buy_live.py` | 开盘买 |
| `strategy_yidong_close_buy_live.py` | 尾盘买 |
| `strategy_yidong_close_sell_live.py` | 尾盘卖 |

三个文件互相独立，不 import 公共模块。

---

## QMT 通用设置

- 周期：**1 分钟**
- 默认资金账号：**30262698**（可在 QMT `ContextInfo.accountid` 覆盖）
- 买入单票上限：**10 万元**（`max_amount_per_stock`，默认 100000）
- 买卖股数：**100 股整数倍**（`min_shares`，默认 100）
- 接口从 `__main__` 取：`passorder` / `cancel` / `get_trade_detail_data`
- 入口：`init` + `handlebar`（别名 `handleBar` / `handle_bar`）
- 仅 `C.is_last_bar()` 时执行
- 回测不发单（`live_orders=False` 或回测上下文）

### 可选 ContextInfo 参数（通用）

| 参数 | 默认 | 说明 |
|------|------|------|
| `live_orders` | True | False 只打日志不下单 |
| `max_retry` | 3 | 撤单/废单后重挂最多次数（首单之外最多再挂 3 次） |
| `quick_trade` | 2 | passorder 快速交易 |
| `strategy_order_name` | 见各策略 | 策略名（备注另有 23 字符上限） |
| `min_shares` | 100 | 最小买卖单位 |

> **撤单等待**：买策略默认 `withdraw_secs=15`；卖策略默认 `withdraw_secs=60`。可在各自策略参数中覆盖。

## 策略1：开盘买

- **handlebar**：09:25–11:30、13:00–15:00（午休 11:30–13:00 不跑）
- **实际下单**：09:25–09:35
- **日志**：板块成分变化时打 `[板块重载]` + `[今日待买池]`（名称+代码）
- **股票池**：板块 `卡异动开盘买入池`（`sector_name` 可覆盖）；**24 小时**每根 K 线重拉板块；**仅成分变化**时打日志
- **单票上限**：10 万，100 股整数倍；**不因已有持仓扣减额度**（允许加仓）
- **涨跌幅过滤**（现价 / 昨收）：
  - 60/00：跌幅 [-10%, -8%] 或涨幅 >8% → 不买
  - 68/30：跌幅 [-30%, -15%] 或涨幅 >15% → 不买
- **限价**：现价 × (1 + **2.5%**)，`buy_premium_pct` 默认 0.025
- **未成交处理**：**15 秒**后检查；挂单未成交则撤单重挂；**废单/已撤也会重挂**；最多 `max_retry` 次
- **重挂溢价**：`buy_premium_retry_step` 默认 **0**（重挂不额外加价）
- **备注前缀**：`YDO_*`（最长 23 字符）

### 可选参数（开盘买）

| 参数 | 默认 |
|------|------|
| `max_amount_per_stock` | 100000 |
| `buy_premium_pct` | 0.025 |
| `buy_premium_retry_step` | 0 |
| `withdraw_secs` | 15 |
| `sector_name` | 卡异动开盘买入池 |

## 策略2：尾盘买

- **handlebar**：09:25–11:30、13:00–15:00（午休不跑）
- **实际下单**：14:55–15:00
- **日志 / 股票池**：同开盘买，`[今日待买池]` 仅在板块变化时打印
- **股票池**：板块 `卡异动尾盘买入池`
- **单票上限 / 涨跌幅 / 加仓**：同开盘买
- **限价**：现价 × (1 + **1%**)，`buy_premium_pct` 默认 0.01
- **未成交处理**：**15 秒**撤单重挂；**废单/已撤也会重挂**；重挂时溢价每轮 +0.5%（`buy_premium_retry_step=0.005`）
- **备注前缀**：`YDCB_*`（代码内 `remark_prefix + 'B'`）

| `buy_premium_retry_step` | 0.005 |
| `withdraw_secs` | 15 |
| `sector_name` | 卡异动尾盘买入池 |

## 策略3：尾盘卖

- **handlebar**：09:25–11:30、13:00–15:00（午休不跑）
- **实际下单**：14:55–15:00
- **数据源**：CSV + 账户持仓（不用板块）
- **CSV 24h 监控**：每根 K 线重读；**仅 CSV 内容变化**时打 `[CSV重载]` + `[今日待卖池]`；持仓变化时打 `[账户持仓]`
- **卖出窗口内**：每根 K 线打 `[监控]`（含持仓交易日、盈亏、是否触发）
- **CSV 路径**（按优先级）：
  1. `ContextInfo.sell_watch_csv_path`
  2. 环境变量 `YIDONG_SELL_WATCH_CSV`
  3. 代码内默认绝对路径：`C:\Users\Dustin.hou\belllla_web\qmt\实盘策略\yidong_regulation_sell_watch.csv`
  4. 策略同目录 `yidong_regulation_sell_watch.csv`
  5. 从 QMT 工作目录（常为 `bin.x64`）向上查找
- **CSV 列**：`股票代码, 开仓日, 触发卖出信号, 卖出股数, 开仓价`
- **编码**：utf-8-sig / gbk / utf-8 自动尝试

### 卖出触发（满足任一即卖）

1. **止盈**：盘中 **09:25–15:00** 现价 ≥ 止盈价即卖（限价 = 止盈价；开仓价×16% 或 CSV 第 6 列）
2. 触发卖出信号 = **0**（G0 强平，**14:55–15:00**）
3. 持仓 **≥ 6 个 A 股交易日**（相对开仓日，节假日不计，`max_hold_days` 默认 6）
4. 持仓 **> 2 个交易日** 且 浮亏 **≥ 8%**（相对成本，`stop_loss_pct` 默认 8）

> 信号 = **1** 时不会因 G0 卖出，但满 6 交易日、止损 8% 仍会触发。

### 下单规则

- 数量：优先读 CSV **卖出股数**（按原数，**不必整百**）；取 `min(CSV股数, 可卖)`；旧表无该列则卖出全部可卖
- CSV 股数 ≤ 0 或无效 → 跳过不卖
- **限价**：现价 × (1 - 1%)，`sell_discount_pct` 默认 0.01
- **60 秒**未成交撤单重挂，废单也会重挂，最多 `max_retry` 次
- **备注前缀**：`YDS_*`

### 可选参数（卖策略）

| 参数 | 默认 |
|------|------|
| `stop_loss_pct` | 8 |
| `max_hold_days` | 6 |
| `sell_discount_pct` | 0.01 |
| `withdraw_secs` | 60 |

## CSV 卖出监控表示例

```csv
股票代码,开仓日,触发卖出信号,卖出股数,开仓价
600000.SH,20250601,1,1000,10.00
000001.SZ,20250601,0,500,8.50
688143.SH,20260529,1,636,10.75
```

- 信号 = **0**：当日尾盘 G0 强平（若可卖）
- 信号 = **1**：继续持有，直至满 6 交易日或触发止损
- **卖出股数**：本笔买入对应数量；同代码多笔请分多行；不填则卖全部可卖
- **开仓价**：该笔实际买入成交价；策略按 `开仓价 × (1 + take_profit_pct%)` 自动算止盈价（默认 **16%**）
- **可选第 6 列止盈价**：若填写则覆盖自动计算；**现价 ≥ 止盈价** 即卖，限价 = 止盈价
- 策略参数 `ContextInfo.take_profit_pct` 可改止盈比例（默认 16）
- **表格编辑**：用 VS Code / Excel 打开时，**D 列=卖出股数、E 列=开仓价**；股数与开仓价分列填写，勿在股数后加逗号
- 说明注释见本文档；CSV 内仅保留表头与数据行，便于表格视图显示
