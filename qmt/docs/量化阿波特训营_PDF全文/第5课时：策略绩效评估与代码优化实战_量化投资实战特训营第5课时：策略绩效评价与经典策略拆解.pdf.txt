总收益 vs 年化收益 - 选对衡量标尺
不同时间跨度下的绩效评估差异
总收益率 (Total Return)
R = (P_end - P_start) / P_start
年化收益率 (Annualized Return)
CAGR = (P_end / P_start)^(1/n) - 1
总收益率：代表完整期间的绝对回报，直观但不含时间成本。
年化收益率：考虑复利效应，便于跨期比较不同生命周期的策略。
应用场景：短期策略（如日内）关注总收益；长期资产配置必须看年化收益。
实例：100% 总收益的年化差异
同样的翻倍收益，在3年内实现年化约 26%，而在5年内实现则降至 15%。
复利的力量：72法则
翻倍所需年数 ≈ 72 / 年化收益率(%)

波动率 - 衡量收益稳定性的关键指标
波动率（标准差）是量化风险的第一道防线
数学定义与公式
σ = √(Σ(r_i - μ)² / N)
反映收益率相对于均值的离散程度。离散度越高，不确定性越大。
Python 实现
# 计算年化波动率
 returns = df['close'].pct_change()
 vol = returns.std() * np.sqrt(252)
print(f"Annual Vol: {vol:.2%}")
注意：波动率小不等于好。高收益策略通常伴随高波动，关键在于收益是否能
覆盖波动风险（夏普比率）。
风险解读 (正态分布假设)
10% 波动率含义：意味着在 68% 的年份中，收益率会
落在
[均值 - 10%, 均值 + 10%] 范围内。
年化方法：日波动率 × √252。使用平方根是因为方差具有时间可加性。
应用要点
衡量策略的“平滑度”。
用于计算风险调整收益（夏普比率）。
作为止损和仓位管理的依据（如 ATR 止损）。
为什么日波动率要乘
以√252而不是252？

夏普比率 - 单位风险的收益
量化投资中最广泛使用的风险调整收益指标
核心公式
Sharpe = (Rₚ - R_f) / σₚ
Rₚ：策略预期年化收益率
R_f：无风险利率（如 10 年期国债收益率）
σₚ：策略收益率的年化波动率（标准差）
# Python 快速计算
 excess_ret = returns.mean() * 252 - rf
 ann_vol = returns.std() * np.sqrt(252)
 sharpe = excess_ret / ann_vol
解读标准
Sharpe < 0
策略失败
0 - 1.0
表现一般
1.0 - 2.0
表现较好
> 2.0
表现优秀
核心认知：夏普比率越高，代表你在承受同等风险的情况下，获得的超额收
益越高。它是投资组合比较的黄金标尺。

===== 第 10 页 =====

夏普比率的陷阱 - 看似完美数据的警示
不要被虚高的 Sharpe Ratio 蒙蔽了双眼
无风险利率选择不当
随意使用 0 或过低的利率作为 Rf，会显著虚增夏普比率，导致策略评价失真。
建议：使用同期 10 年期国债收益率或 SHIBOR。
样本期间太短
短期（如 1 年内）的夏普比率受偶然因素影响巨大，不具备统计学意义上的
稳定性。
建议：至少使用 3 年以上的历史数据进行评估。
非正态分布失效
夏普比率假设收益率呈正态分布。若存在“肥尾”或严重偏离，波动率无法准
确衡量风险。
方案：使用罗伊安全比率或索提诺比率作为补充。
过拟合风险 (Sharpe > 3)
在回测中出现 Sharpe > 3.0 通常意味着过度优化参数，在样本外极易失效。
警示：警惕“太好而不可信”的数据，加强样本外验证。
核心提示：夏普比率只是一个参考维度。一个 Sharpe 为 1.5 但回撤可控的策略，往往比 Sharpe 为 3.0 但逻辑脆弱的策略更具实战价值。
概率夏普比率（PSR）：
量化夏普比率的可信度

===== 第 11 页 =====

超越夏普 - 更精准的风险调整收益指标
针对下行风险与主动管理能力的深度评估
索提诺比率 (Sortino Ratio)
Sortino = (Rₚ - R_f) / σ_down
核心差异：只计算下行波动率（低于目标收益的波动），忽略上行波动。
人性化：更符合投资者心理——我们只讨厌亏损，不讨厌超额盈利带来的波动。
适用场景：对亏损极度敏感的策略，或收益分布不对称的策略。
信息比率 (Information Ratio)
IR = (Rₚ - R_b) / σ(Rₚ - R_b)
核心差异：衡量超额收益相对于跟踪误差（主动风险）的稳定性。
能力评估：反映基金经理或策略在偏离基准时，获取超额收益的能力。
适用场景：指数增强策略、主动管理型基金的绩效评价。

交易质量 - 胜率 vs 盈亏比的博弈
理解数学期望是交易盈利的基石
数学期望公式 (Expected Value)
E = W × P_avg - (1 - W) × L_avg
•W (Win Rate) ：胜率 = 赚钱交易数 / 总交易数
•P_avg ：平均盈利金额
•L_avg ：平均亏损金额
•盈亏比= P_avg / L_avg
核心认知：高胜率并不代表高收益。许多趋势跟踪策略胜率不足 40%，但凭借极高的盈
亏比（如 3:1 以上）依然能获得丰厚回报。

class MACDStrategy(bt.Strategy):
    # 1. 参数化设计：方便后续调优
    params = (
        ('fast', 12),
        ('slow', 26),
        ('signal', 9),
    )
    def __init__(self):
        # 2. 指标初始化：使用内置指标
        self.macd = bt.ind.MACD(
            self.data.close,
            period_me1=self.params.fast,
            period_me2=self.params.slow,
            period_signal=self.params.signal
        )
 # 3. 信号逻辑：使用 CrossOver 避免浮点数比较误差
        # crossover > 0 表示金叉 (macd 上穿 signal)
        # crossover < 0 表示死叉 (macd 下穿 signal)
        self.crossover = bt.ind.CrossOver(self.macd.macd, 
self.macd.signal)
    def next(self):
        # 4. 交易执行
        if not self.position:
            if self.crossover > 0:
                self.buy() # 全仓买入
        elif self.crossover < 0:
            self.close() # 清仓卖出

为什么风险加倍，收益反而缩水？
这就涉及到了量化交易中一个极易被忽视的概念：波动率拖累 (Volatility 
Drag)。
1. 数学上的非对称性：
亏损 10% 需要涨 11% 回本。
亏损 40% 需要涨 67% 回本。
当风险度提高，回撤幅度加深（从 -38% 到 -40%），资金在高位被剧烈"削峰"，导致在随后的反弹中本金基数过小，无法产
生足够的复利效应。
2. 破产风险 (Risk of Ruin)：
2% 的风险看似不大，但在连续亏损期（如 BTC 熊市），连续 5 次止损就会导致本金缩水约 10%。
实盘数据告诉我们：最佳的风险敞口 (Optimal F) 往往比直觉要小得多。在 BTC 这种高波动的资产上，1% 甚至 0.5% 的风险敞
口可能才是长久生存之道。

定义：回测时只包含目前仍上市的股票，剔除了历史上已退市的失败者。
后果：回测结果会因为“自动避开了雷区”而产生显著的虚高。
数据窥探 (Data Snooping)
测试 10,000 个随机规则
“发现”一个表现完美的规律
定义：通过穷举大量随机组合，纯靠概率在历史噪音中“撞”到一个好结果。
本质：这种“规律”在未来几乎不可能重演，因为它只是历史数据的偶然。
使用包含退市股的全样本库
严格的样本外 (OOS) 验证
坚持逻辑驱动而非数据驱动

• 中轨 (Mid) = N 日简单移动平均 (SMA)
 • 上轨 (Upper) = 中轨 + k × N 日标准差
 • 下轨 (Lower) = 中轨 - k × N 日标准差
* 常用参数：N=20, k=2

在震荡市中，触碰下轨买入，触碰上
轨卖出。相信价格会回到均值。
模式 B：趋势突破
在宽幅波动中，价格突破上轨买入。
相信趋势正在形成。
开口扩大：波动率增加，趋势可能启动。
开口缩小：波动率降低，市场进入盘整。

Vectorized Momentum
Implementation
import pandas
as pd
import numpy
as np
def
momentum_strategy
(df, lookback=
252 ):
# 1. 计算过去 N 天的收益率 (动量值)
df[ 'mom' ] = df[ 'close' ].pct_change(lookback)
# 2. 生成信号：动量为正买入，否则持币
df[
'signal' ] = np.where(df[
'mom' ] > 0, 1, 0)
# 3. 计算策略收益 (注意：信号需 shift(1) 避免未来函数)
df[
'ret' ] = df[ 'close' ].pct_change()
df[
'strat_ret' ] = df[ 'signal' ].shift( 1) * df[ 'ret' ]
# 4. 计算累积净值
df[
'cum_ret' ] = ( 1+ df[ 'strat_ret' ]).cumprod()
return df

基于 RSI 指标的超买超卖反转逻辑：捕捉短期情绪波动的极值
RSI Reversal
Implementation
def
rsi_reversal_strategy
(df, period=
14 , low= 30 , high= 70 ):
# 1. 计算 RSI 指标
delta = df[
'close' ].diff()
gain = (delta.where(delta >
0, 0)).rolling(period).mean()
loss = (-delta.where(delta <
0, 0)).rolling(period).mean()
rs = gain / loss
df[ 'rsi' ] = 100 - ( 100 / ( 1+ rs))
# 2. 生成信号：超卖买入，超买卖出
df[
'signal' ] = 0
df.loc[df[
'rsi' ] < low,
'signal' ] = 1
df.loc[df[
'rsi' ] > high,
'signal' ] = - 1
# 3. 计算收益 (注意 shift(1))
df[
'strat_ret' ] = df[ 'signal' ].shift( 1) * df[ 'close' ].pct_change()
return df

风险点：接飞刀

动量 + 反转的配置方法 - 动态平衡
如何科学分配资金：从简单权重到风险平价的进阶之路
1. 固定权重 (Fixed Weight)
最简单的方法，如 50% 动量 + 50% 反转。优点是实现容易，缺点是未考虑不同策略的风险特征差异。
2. 风险平价 (Risk Parity)
