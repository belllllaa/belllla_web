FRAME 回测框架

结果部分 (Results)

观察多头组（Group 1）到空头组（Group 10）的收益是否具有单调性。单调性

（如2015、2018）下的表现。

===== 第 13 页 =====

研报中的常见陷阱
幸存者偏差 (Survivorship Bias)
什么是幸存者偏差？
在回测过程中，如果只使用“当前仍在上市”的股票作为历史样本，而
忽略了历史上已经退市或被长期停牌的股票，就会产生幸存者偏差。
核心危害
退市股票通常表现极差。剔除它们会显著虚增历史回测收益率，误导
投资者认为策略在历史上非常稳健。
如何识别？
检查研报的“样本空间”说明。如果提到“以当前中证500成分股为样本
回测过去10年”，则必然存在幸存者偏差。正确的做法是使用“历年成
分股”。
实战案例：
某小市值策略研报，回测2010-2020年收益。由于未包含期间退
市的200余只垃圾股，其年化收益比实际高出约4.5%，且最大回
撤被低估了12%。

未来函数 (Look-ahead Bias)
在计算因子或进行回测时，使用了在“当前时间点”尚不可获得的信息。这
是量化研究中最严重的逻辑错误。
场景
错误做法 (未来函数)
正确做法
调仓价格
T日收盘价计算，T日收盘买
入
T日收盘计算，T+1开盘买
入
财务数据
使用财报所属季度的末日
使用财报实际披露日期
高送转
预案公告前就计入除权预期
公告日后才计入信息
识别与案例分析
如何识别： 重点检查代码中 shift(1) 的使用。如果因子值与收益率在同一
行，且没有时间偏移，极大概率存在未来函数。
案例：2-3倍IC虚高陷阱
某研报宣称其动量因子月度IC高达15%。复现发现，其回测逻辑是“月底
最后1分钟计算因子并按当日收盘价成交”。
修正后： 改为次日开盘成交，IC骤降至4.2%。这消失的10%就是典型的
“偷看未来”带来的虚假收益。

===== 第 15 页 =====

研报中的常见陷阱
数据窥探 (Data Snooping)
什么是数据窥探？
通过对同一数据集进行成百上千次的重复测试，最终筛选出表现最好的那
一个参数。这本质上是在“挖掘”随机噪声而非真实规律。
隐蔽性分析
它不像未来函数那样有明显的逻辑错误，回测代码本身可能是完全正确的。
它利用了统计学上的“多重比较”谬误。
如何识别？
如果研报中使用的参数非常“奇特”（如：23日均线、1.27倍标准差），且
没有合理的经济学解释，通常就是数据窥探的结果。
“如果你对数据严刑拷打足够久，它总会招供的。” —— 罗纳德·科斯
防范建议
坚持使用行业通用的“整数”参数（如5, 10, 20），并进行严格的样本外测
试（Out-of-Sample Testing）。

===== 第 16 页 =====

研报中的常见陷阱
参数过拟合 (Overfitting)
过拟合的典型标志
参数极其敏感：窗口期从20天改到21天，收益下降一半。
样本内外差异：2010-2018年净值完美，2019年后大幅亏损。
复杂度过高：因子公式套了5层以上的非线性变换。
危害： 过拟合的策略本质上是在刻舟求剑。它捕捉的是特定历史时期的“噪
音”，在未来的实盘中几乎必然失效。
三个改进方向
增加样本量
跨越更长的历史周期，覆盖不同的牛熊市场环境。
奥卡姆剃刀原则
尽量减少参数数量。如果两个参数能达到三个参数的效果，选两个。
引入稳健性约束
在参数空间内进行平滑处理，确保参数在一定范围内变动时效果稳
定。

===== 第 32 页 =====

研究动机 Part 2：威廉指标的改进逻辑
威廉指标 (WMS) 的定义
威廉指标反映了当前收盘价在过去 $N$ 个交易日价格区间（最高价
与最低价之间）的相对位置。
$WMS = \frac{H_N - C}{H_N - L_N} \times 100$
其中 $H_N$ 和 $L_N$ 分别为过去 $N$ 日内的最高价和最低价。
为什么引入威廉指标？
打破“单日”限制
蜡烛图影线仅关注当日内的价格波动，而威廉指标影线捕捉了跨
日的价格压力分布。
处理跳空缺口
在出现大幅跳空时，蜡烛图影线可能失效，而威廉指标通过参考
历史区间保持了逻辑的一致性。

性能评估指标 Part 1：IC、RankIC 与 ICIR
IC (Information Coefficient)
$IC = Corr(Factor_t, Return_{t+1})$
衡量因子值与下期收益率之间的线性相关程度。 
RankIC： 将因子值和收益率转换为排名后计算相关系数。相比 IC，
RankIC 对异常值更稳健，是量化研究中最常用的指标。
ICIR (IC Information Ratio)
$ICIR = \frac{Mean(IC_{series})}{Std(IC_{series})}$
衡量因子收益的稳定性。 
 如果一个因子 IC 均值很高但波动巨大，其 ICIR 会较低。优秀的因子不
仅要求“赚得多”，更要求“赚得稳”。通常 ICIR > 0.5 视为表现良好。
评判准则： IC 衡量的是“深度”（预测能力），ICIR 衡量的是“广度”（预测的稳定性）。二者结合构成了评价因子的第一道门槛。

风格中性化 Part 1：必要性与回归法原理
为什么要中性化？
剥离已知风险因素
许多价量因子天然带有“小市值”或“高波动”偏好。中性化
能确保我们赚的是因子的 Alpha，而非风格的 Beta。
提升因子稳健性
中性化后的因子在不同风格轮动（如大盘价值 vs 小盘成
长）的市场环境下表现更稳定。
降低组合相关性
中性化有助于将新因子与已有因子库解耦，提升多因子
组合的边际贡献。
回归法中性化原理示意图
残差 (Residual) = 纯净 Alpha
$Factor_{raw} = \beta \cdot Style + \epsilon$
回归后的残差 $\epsilon$ 即为剥离了风格因素后的“中性化因子”。

deffilter_stocks(df, info_df):
    # 1. 排除 ST 股
    df = df[~df['name'].str.contains('ST')]
    
    # 2. 排除上市不足 3 个月的新股# info_df 包含 list_date 字段

    df['days_since_list'] = (df.index.get_level_values('date') - 
                             pd.to_datetime(df['list_date'])).dt.days
    df = df[df['days_since_list'] > 90]
    
    # 3. 排除停牌股 (成交量为 0)
    df = df[df['volume'] > 0]
    
    return df.drop(columns=['list_date', 'days_since_list'])

威廉指标计算代码
威廉指标说明
- 公式: WR = (Close - Low) / (High - Low) × 100
- 含义: 当前收盘价在最高最低价区间中的相对位置
- 取值范围: 0-100
  - 接近100：收盘价接近最高价（超买）
  - 接近0：收盘价接近最低价（超卖）
- 短期vs长期:
  - `wr_s` (5日): 捕捉短期价格动能
  - `wr_l` (20日): 反映中期价格趋势
  - `wr_trend`: 短期-长期，判断动能转折
# 4. 计算威廉指标（Williams %R）

df['wr_l'] = grouped['wr'].rolling(self.wr_window_long).mean().reset_index(level=0, 
drop=True)   # 20日
# 计算WR指标的趋势因子 (短期WR - 长期WR)
df['wr_trend'] = df['wr_s'] - df['wr_l']

def _clip_outliers(self, df: pd.DataFrame, cols: List[str]) -> pd.DataFrame:
    """根据指定的去极值方法对DataFrame的指定列进行去极值处理"""
    if df.empty:
        return df
    method = (self.outlier_method or '').lower()
    for col in cols:
        if col not in df.columns:
            continue
        series = df[col].astype(float)
        if method == 'quantile':  # 分位数去极值
            q = float(self.outlier_param)  # 例如0.01表示截去上下1%
            lower = series.quantile(q)
            upper = series.quantile(1 - q)
            df[col] = series.clip(lower=lower, upper=upper)
        else:  # 默认使用 Sigma (标准差) 去极值
            n_sigma = float(self.outlier_param)  # 例如3.0表示±3σ
            mean = series.mean()
            std = series.std()
            if std == 0 or np.isnan(std):
                continue
            df[col] = series.clip(lower=mean - n_sigma * std,
                                  upper=mean + n_sigma * std)
    return df

- 方法：`sigma`（标准差法）
- 参数：`3.0`（3倍标准差）
- 效果：截断超出均值±3σ范围的极端值

@staticmethod
def _zscore(series: pd.Series) -> pd.Series:
    """计算 Series 的 Z-score 标准化"""
    s = series.astype(float)
    std = s.std()
    if std == 0 or np.isnan(std):
        return s * 0.0
    return (s - s.mean()) / (std + 1e-8)

其中：
- X: 原始因子值
- μ: 均值
- σ: 标准差
- 1e-8: 防止除零的微小常数
公式: Z = (X - μ) / σ

市值中性化：残差计算与错误案例
正确做法总结
1. 必须包含常数项 (Intercept)
如果不加常数项，回归线强制过原点，得到的残差均值不为 0，会破坏后续标
准化的逻辑。
2. 必须按期 (Cross-sectional)
市值与因子的关系随时间变化。全样本回归会引入未来信息，导致回测失效。
典型错误案例
错误 1：全样本回归 (Global Regression)
将过去 10 年的所有数据放在一起跑一个回归。这会导致 2010 年的因子
值被 2020 年的市值分布所污染，属于严重的逻辑错误。
错误 2：忽略行业中性化
仅对市值中性化，忽略了行业暴露。如果因子在某行业（如计算机）
表现极好，不进行行业中性化会导致组合在特定行业过度集中。

defprepare_backtest_data(df):

defcalculate_group_returns(df, n_groups=10):
    # 1. 在每一期内进行分组
    df['group'] = df.groupby('date')['factor'].transform(
        lambda x: pd.qcut(x, n_groups, labels=False) + 1
    )

    # 3. 转换为透视表格式 (日期为行，组别为列)
    group_ret_table = group_ret.unstack()
    
    return group_ret_table

defassemble_ls_return(group_ret_table, is_negative=True):
    # 1. 提取多头和空头组if is_negative:

指标计算：IC 与 RankIC 的实现
IC 计算的核心逻辑
IC 是衡量因子预测能力最直接的指标。在代码实现中，
我们需要在每一个调仓日（横截面）上计算因子值与下
期收益率的相关性。
RankIC 的优势：
RankIC 使用 Spearman 相关系数，只关注排名，能够
有效过滤掉收益率分布中的极端异常值，结果更稳健。
defcalculate_ic_series(df):
    # 1. 计算 Pearson IC (线性相关)
    ic_series = df.groupby('date').apply(
        lambda x: x['factor'].corr(x['target_ret'])
    )
    
    # 2. 计算 RankIC (Spearman 排名相关)
    rank_ic_series = df.groupby('date').apply(
        lambda x: x['factor'].corr(x['target_ret'], method='spearman')
    )
    
    return ic_series, rank_ic_series

指标计算：ICIR 与稳定性指标
稳定性评估逻辑
ICIR 是衡量因子稳定性的核心指标。它排除了运气的成分，
反映了因子在不同月份表现的一致性。
其他稳定性指标：
IC 胜率：IC > 0 的月份占比。
IC t 统计量：检验 IC 是否显著异于 0。
defcalculate_ic_metrics(ic_series):
    # 1. 计算 IC 均值
    ic_mean = ic_series.mean()
    
    # 2. 计算 IC 标准差
    ic_std = ic_series.std()
    
    # 3. 计算 ICIR
    icir = ic_mean / ic_std
    
    # 4. 计算 IC 胜率 (假设因子为正向)
    ic_win_rate = (ic_series > 0).mean()
    
    return {
        'IC Mean': ic_mean,
        'ICIR': icir,
        'IC WinRate': ic_win_rate
    }
# 示例调用
metrics = calculate_ic_metrics(rank_ic)

指标计算：年化收益与最大回撤
风险与收益评估
在多空回测中，我们需要将月度收益转化为年化指标，
并识别历史上最极端的亏损情况。
核心逻辑：
最大回撤：净值曲线与其历史最高点的最大偏离。
年化收益：考虑复利效应的平均年度回报。
defcalculate_risk_return(ls_ret_series):
    # 1. 计算累积净值
    nav = (ls_ret_series + 1).cumprod()
    
    # 2. 计算年化收益 (假设月频数据)
    total_ret = nav.iloc[-1] - 1
    n_months = len(ls_ret_series)
    ann_ret = (total_ret + 1) ** (12 / n_months) - 1# 3. 计算最大回撤
    cum_max = nav.cummax()
    drawdown = (nav / cum_max) - 1
    max_dd = drawdown.min()
    
    return {
        'Ann Return': ann_ret,
        'Max Drawdown': max_dd
    }

致命逻辑错误：维度混淆
全样本标准化 (Global Z-Score)

横截面标准化 (正确)
IC 均值 = 3.2%
真实反映因子预测能力
全样本标准化 (错误)
IC 均值 = 12.5%
虚高 4 倍，源于未来信息泄露
警示：如果你在复现时发现 IC 异常高，优先检查预处理的维度是否
正确。

预处理方法差异：去极值方法的影响对比
常用去极值方法
3-Sigma 方法
基于均值和标准差。缺点是均值本身易受极端值拉低或抬高，导致识别
不准。
MAD 方法 (推荐)
使用中位数绝对偏差。由于中位数非常稳健，MAD 是目前量化界最主流
的选择。
分位数截断 (Quantile)
直接将因子值排名前后 1% 的数据强制拉回边界。简单粗暴但有效。
不同方法对 IC 的扰动
以 UBL 因子为例，不同去极值方法下的复现结果：
使用 MAD (研报默认)
IC = -4.30%
使用 3-Sigma
IC = -4.18%
使用 1% 分位数截断
IC = -4.25%
不进行去极值
IC = -3.85%
结论： 去极值方法会导致 0.1% - 0.4% 的 IC 偏差。若研报未说明，优先
尝试 MAD 方法。

预处理方法差异：中性化顺序与信息流失
推荐的预处理顺序
1. 去极值 (Winsorize)
2. 市值/行业中性化
3. 横截面标准化 (Z-Score)
逻辑： 先去极值是为了防止异常值干扰回归系数；最后标准化是为了确保各
因子在融合前具有相同的量纲。
顺序错误的影响
信息流失风险
如果先标准化再中性化，回归后的残差均值不再为 0，且标准差不再为 1，
会导致因子在融合时权重失衡。
风格“回潮”
若在中性化后再进行复杂的非线性变换，可能会重新引入已被剥离的市值
相关性，使中性化功亏一篑。
复现差异估计
预处理顺序的微调通常会导致 IC 指标出现 10% - 20% 的相对偏差。

敏感性分析：参数交互作用分析
参数间的“博弈”
敏感度 vs 平滑度
标准化窗口 $M$ 越小，因子对近期情绪越敏感；特征提取窗口 $T$ 越大，
因子越平滑。二者共同决定了策略的换手率。
参数高原的意义
如果最优 IR 出现在矩阵中心（如 $T=20, M=5$），且周围参数表现相近，
说明该逻辑具有极强的泛化能力。
规避“参数孤岛”
若某个参数组合表现极好，但稍微变动一点性能就骤降，说明该点是由于
过拟合产生的随机噪声。
实际应用建议
如何选择最终参数？
优先选择“高原”中心：不要选最高点，而要选最稳健的区域中心。
考虑换手成本：如果两个组合 IR 相近，优先选择 $T$ 较大的组合以
降低换手。
样本外压力测试：在确定参数后，必须在未参与测试的数据上进行
最终验证。
* 记住：在量化交易中，稳健性（Robustness）永远优于名义收益率。

稳健性评价：参数敏感 vs 稳健的量化区分
量化判断标准
1. 变异系数 (CV) 准则
在参数空间内，年化收益率的变异系数（标准差 / 均值）应 < 15%。
CV 越小，稳健性越高。
2. 趋势单调性
绩效指标随参数的变化应呈现平滑的单峰或单调趋势，严禁出现剧烈
的随机跳动（锯齿状）。
3. 样本外回撤一致性
样本外（OOS）的最大回撤幅度不应超过样本内（IS）的 1.5 倍。

问题诊断：严重程度等级与修复指南
严重程度定义
致命 (Critical)： 逻辑错误，如未来函数或方向反向。
必须立即停止并修复。
严重 (Major)： 显著影响收益或风险，如成本漏算。
需重点排查。
轻微 (Minor)： 数值扰动，如去极值方法差异。了解
原因即可，不影响逻辑。
复现差异修复指南
1
隔离变量 (Isolate Variables)
每次只修改一个参数（如仅修改手续费），观察结果变化，确定差异的单一来
源。
2
逐行对标 (Row-by-row Check)
选取一只典型股票，打印其因子的中间计算步骤，与研报中的逻辑进行人工核
对。
3
敏感性回溯 (Sensitivity Backtrack)
如果整体不匹配，尝试在参数空间内进行小范围搜索，看是否能找到与研报吻
合的隐藏参数。
