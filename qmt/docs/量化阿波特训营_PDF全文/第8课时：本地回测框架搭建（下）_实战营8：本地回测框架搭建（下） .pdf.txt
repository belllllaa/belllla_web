持仓管理的核心概念
什么是持仓管理?
关键术语:
- 持仓(Position):当前账户持有的股票列表及其数量
- 调仓(Rebalance):根据新的信号,卖出旧股票、买入新股票
- 换手率(Turnover):衡量调仓频率的指标
调仓逻辑示例:
换手率计算:
换手率 = (买入数量 + 卖出数量) / (2 × 原持仓数量)
示例:
原持仓5只,卖出2只,买入2只
换手率 = (2 + 2) / (2 × 5) = 40%
今日持仓: [A, B, C, D, E]
新信号:   [B, C, F, G, H]
需要卖出: [A, D, E]  ← 旧持仓中不在新信号里的
需要买入: [F, G, H]  ← 新信号中不在旧持仓里的
保持不变: [B, C]     ← 交集

===== 第 8 页 =====

交易成本的构成
真实的交易成本 - 三大成本组成:
1. 佣金(Commission)
- 买卖双向收取
- 常见费率:万2.5 ~ 万3(0.00025 ~ 0.0003)
- 最低5元(小额交易吃亏)
3. 印花税(Stamp Duty)
2. 滑点(Slippage)
- 买入时价格略高,卖出时价格略低
- 市价单成交价与预期价差
- 估计值:0.1%(大单可能更高)
仅卖出时收取
- 固定费率:0.1%(千分之一)
- 国家税收,无法避免
总成本公式:
买入成本 = 佣金 + 滑点
卖出成本 = 佣金 + 滑点 + 印花税
单向成本 ≈ 0.03% + 0.1% = 0.13%
双向成本 ≈ 0.13% × 2 + 0.1% = 0.36%

===== 第 9 页 =====

交易成本的实际影响
高频交易的成本陷阱
场景 1:每天调仓(换手率 100%)
初始资金:100万
每天成本:100万 × 100% × 0.36% = 3,600元
一年成本:3,600元 × 250天 = 900,000元 = 90万!
结论:一年成本吃掉 90% 本金,策略几乎必败!
场景 2:每5天调仓(换手率 20%)
初始资金:100万
每次成本:100万 × 20% × 0.36% = 720元
一年成本:720元 × 50次 = 36,000元 = 3.6万
结论:成本占比 3.6%,可以接受
场景 3:每月调仓(换手率 5%)
初始资金:100万
每次成本:100万 × 5% × 0.36% = 180元
一年成本:180元 × 12次 = 2,160元
结论:成本极低,但可能错过短期机会
最佳实践:
- 中低频因子:每5-10天调仓
- 日内因子:需要超强选股能力才能覆盖成本

===== 第 10 页 =====

最大回撤(Max Drawdown)
风险的终极指标
定义:
资产净值从最高点到后续最低点的最大跌幅
计算步骤:
# 1. 累计净值曲线
cumulative_returns = [1.0, 1.02, 1.05, 1.03, 0.98, 1.01, 
1.04]
# 2. 计算历史最高净值
running_max = [1.0, 1.02, 1.05, 1.05, 1.05, 1.05, 1.05]
# 3. 回撤率 = (当前净值 - 历史最高) / 历史最高
drawdown = [0.0, 0.0, 0.0, -1.9%, -6.7%, -3.8%, -0.95%]
# 4. 最大回撤
max_drawdown = max(abs(drawdown)) = 6.7%
评判标准:
- ✅  优秀策略:最大回撤 < 10%
- ⚠  可接受:最大回撤 < 20%
- ❌  高风险:最大回撤 > 30%
心理意义:
- 投资者最关心的指标(比收益率更重要)
- 决定了策略能否长期运行
- 机构通常设置 20-25% 的强制平仓线

===== 第 11 页 =====

pandas.cummax() 函数详解
计算历史最高值的关键函数
函数作用:
equity_curve.cummax()  # 返回累计最大值序列
在回撤计算中的应用:
# 计算最大回撤的3步
equity_curve = pd.Series([1.0, 1.02, 1.05, 1.03, 0.98])
running_max = equity_curve.cummax()      # [1.0, 1.02, 
1.05, 1.05, 1.05]
drawdown = (equity_curve - running_max) / 
running_max
max_drawdown = drawdown.min()            # -6.7%
关键特性:
- 单调递增: cummax() 永远不会下降
- 高效: O(n) 时间复杂度
直观示例:
净值序列:        [1.0,    1.5,    1.2,    1.8,    1.4]
cummax结果:   [1.0,    1.5,    1.5,    1.8,    1.8]
                           ↑     ↑      ↑     ↑      ↑
                       保持  新高  保持  新高  保持
每个位置的值是从开始到当前位置的最大值
- 常用于计算回撤时的历史最高净值

夏普比率(Sharpe Ratio)
风险调整后的收益
定义:
每承担1单位风险,能获得多少超额收益
示例计算:
年化收益: 15%
无风险利率: 3%(国债收益率)
年化波动率: 12%
Sharpe = (15% - 3%) / 12% = 1.0
评判标准:
- ⭐⭐⭐ Sharpe > 2.0:优秀策略(顶级私募水平)
- ⭐⭐ Sharpe > 1.0:良好策略(大多数量化基金)
- ⭐ Sharpe > 0.5:可接受(勉强跑赢市场)
- ❌ Sharpe < 0:不如无风险资产(买国债更好)
公式:
Sharpe = (策略年化收益 - 无风险利率) / 策略年
化波动率
            = (E[Rp] - Rf) / σp

公式:
Calmar = 年化收益率 / 最大回撤

===== 第 16 页 =====

IC 分析(Information Coefficient)
因子评价的金标准
什么是 IC?
- 定义:因子值与下期收益率的相关系数
- 目的:直接衡量因子的预测能力
- 优势:与市场涨跌解耦,纯粹评估选股能力
评判标准:
- ⭐⭐⭐  |IC| > 0.03:强因子
- ⭐⭐  |IC| > 0.01:有效因子
- ⭐  |IC| > 0.005:弱因子
- ❌  |IC| < 0.005:无效因子
IR(Information Ratio):
IR = Mean(IC) / Std(IC)
意义:IC 的稳定性
- IR > 1.0:稳定因子
- IR < 0.5:不稳定
计算方法(每日一次):
# 某日的因子值
factor_values = [0.05, -0.02, 0.08, ...]  # 3000只股票
# 次日的收益率
next_returns = [0.01, -0.01, 0.02, ...]
# 计算相关系数(通常用 Spearman)
IC = correlation(factor_values, next_returns)

===== 第 17 页 =====

IC 的实战意义
为什么 IC 比收益率更重要?
场景 1:高收益但低 IC
策略收益:50%
IC 均值:0.002
IR:0.1
诊断:收益来自市场 Beta(如牛市),非因子 Alpha
风险:熊市时策略会失效
IC 的四大优势:
1. 市场中性:牛市熊市都能评估
2. 可比性强:不同因子、不同时期可直接对比
3. 风险预警:IC 突然下降说明因子失效
4. 组合构建:多因子可按 IC 加权
IC 胜率:
IC 胜率 = IC > 0 的天数 / 总天数
- IC 胜率 > 60%:优秀
- IC 胜率 > 55%:良好
- IC 胜率 ≈ 50%:随机(无预测能力)
公式:
策略收益:15%
IC 均值:0.025
IR:1.5
诊断:因子有真实选股能力,收益稳定
优势:穿越牛熊,长期有效

分组收益分析(Quintile Analysis)
直观看因子的选股能力
什么是分组收益?
- 将股票按因子值分成N组(通常10组或5组)
- 计算每组的平均收益率
- 观察收益是否单调递增 → 因子有效
评判标准:
- ✅ 单调递增:收益从低到高,因子有效(超级因子)
- ⚠ 大致递增:有些波动但趋势向上,因子还不错
- ❌ 无明显趋势:收益杂乱,因子失效
- ❌ 反向单调:低因子值反而收益高,因子反向
简单示例(分5组):
因子值排序:  [最小] → ① ② ③ ④ ⑤ → [最大]
所有股票:    A,B,C | D,E,F | G,H | I,J,K,L | M,N,O...
组①平均收益:  -2.5%
组②平均收益:  -0.5%
组③平均收益:   1.2%
组④平均收益:   3.1%
组⑤平均收益:   5.8%  ← 因子值最大的组收益最高!

Strategy 基类设计
策略接口抽象
为什么需要 Strategy 基类?
1. 统一接口:所有策略都有相同的方法签名
2. 快速扩展:添加新策略只需继承基类
3. 批量测试:遍历策略列表,自动回测
抽象基类(ABC):
from abc import ABC, abstractmethod
class Strategy(ABC):
    def __init__(self, name: str):
        self.name = name
    @abstractmethod
    def calculate_factor(self, date, data_loader, **kwargs):
        """子类必须实现:计算因子值"""
        pass
    @abstractmethod
    def generate_signal(self, factor_df, top_n):
        """子类必须实现:生成选股信号"""
        pass

# 定义策略列表
strategies = [
    MomentumStrategy(period=20),
    MomentumStrategy(period=60),
    ReversalStrategy(period=5),
    ReversalStrategy(period=10)
]
# 遍历测试
results = []
for strategy in strategies:
    print(f"测试策略: {strategy.name}")
    # 统一的接口调用
    report = engine.run_strategy(strategy, ...)
    results.append({
        'name': strategy.name,
        'sharpe': report['sharpe_ratio'],
        'max_dd': report['max_drawdown'],
        'ic_mean': report['ic_mean']
    })
# 排序找出最优策略
best_strategy = max(results, key=lambda x: x['sharpe'])
print(f"最优策略: {best_strategy['name']}")
